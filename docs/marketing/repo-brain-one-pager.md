# Shibanshu Markdown Viewer: The Repo Brain

## The Big Idea

AI coding tools are powerful, but they usually enter a large repo blind. They open files one by one, fill the context window with raw text, and then answer from an incomplete mental model. A lot of wrong output does not start with bad reasoning. It starts earlier: the model read the wrong files, missed the bridge file, ignored the hub, or ran out of usable context before it understood the system.

Shibanshu Markdown Viewer turns a repo, Claude.md, and Markdown knowledge base into a compact visual brain: hubs, bridges, backlinks, orphans, reading routes, and AI-ready context. The user opens a folder, sees the map, understands the structure, and exports the right context for Claude, ChatGPT, Cursor, or any assistant.

## Why This Is Big

Large repos are now too big for manual reading and too expensive for brute-force AI reading. The market is moving from "chat with files" to "give the model a working map." That is the leap: not another Markdown viewer, but a local context engine that makes both humans and AI understand a codebase faster.

The product creates value in four places:

- Fewer wrong answers: the model starts from the files that matter, not random search.
- Lower AI cost per run: the user stops paying for repeated file exploration.
- Faster onboarding: new engineers and AI agents can understand repo structure quickly.
- Better thinking: the graph exposes hubs, bridges, dead zones, documentation gaps, and reading order.

## Reason To Believe

The local benchmark already shows the shape of the opportunity:

- 924 files: 2,359,465 raw tokens reduced to a 10,812-token map.
- 218:1 compression on large repo context.
- 200 files: 1,057,310 raw tokens reduced to 14,613 map tokens.
- The workflow changes from "read 300 files" to "read the 3 to 10 files the map identifies."

That matters because every AI run has a context tax. If a developer asks the model to fix something and the model burns most of the run rediscovering the repo, the user pays in tokens, time, and trust. A map paid once can guide many future runs.

## Why It Can Feel Groundbreaking

Obsidian made personal notes feel spatial. Cursor made code feel conversational. This sits between them: a local, offline repo brain for humans and AI agents.

The strong version is not a document viewer with a graph button. It is a mental-model layer:

- It reads Markdown, Claude.md, docs, and repo files.
- It builds a 2D/3D graph of relationships.
- It detects hubs, bridges, orphaned knowledge, and likely reading routes.
- It exports compact context maps that LLMs can actually use.
- It lets a person visually navigate complexity before asking the model to act.

## The Market Hook

Every company adopting AI coding assistants is discovering the same pain: the assistant is smart, but the repo is bigger than its working memory. The expensive failure is not just token spend. It is a wrong patch, a wasted review, a hallucinated architecture claim, or a team member losing an hour correcting context.

The line is simple:

**Stop paying AI to get lost. Open your repo's brain.**
